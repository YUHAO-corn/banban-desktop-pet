# BanBan - 情绪守护桌面宠物

> 一只像素小猫，安静地陪你工作。它看得见你的疲惫，会在你需要的时候轻轻提醒你休息。

---

## 产品背景

有些人（尤其是述情障碍者）很难察觉自己正在积累压力。职场教练能通过观察面部紧张来提前干预，但教练不可能一直在。

BanBan 试图扮演这个角色：通过摄像头本地分析面部紧张度，在压力持续偏高时，以桌面宠物的方式温和地提醒用户休息。

核心理念：不是"系统在监控你"，而是"一个小伙伴在关心你"。

---

## 产品定位

- 形态：macOS 桌面宠物应用（后台检测 + Electron 悬浮窗）
- 平台：macOS（内置摄像头）
- 隐私：100% 本地运行，无云端调用，无图像存储

---

## 整体架构

数据从底层检测流向上层表现，是一条单向链路：

```
┌─────────────────────────────────────────────────────────┐
│                       用户的桌面                          │
│                                                         │
│  ① Python 检测引擎（后台进程，已有）                       │
│     摄像头 → MediaPipe → 压力分数                         │
│         │                                               │
│         │ 每2秒写入                                      │
│         ▼                                               │
│  ② /tmp/banban_stress.json（中转文件）                    │
│         │                                               │
│         │ 每2秒读取                                      │
│         ▼                                               │
│  ③ Electron 宿主（待建，核心中间层）                       │
│     - 读取压力数据                                       │
│     - 运行提醒状态机（30秒触发/1分钟超时/5分钟冷却）        │
│     - 管理透明窗口 + 置顶 + 点击穿透                      │
│         │                                               │
│         │ 调用 JS API：setStress(72)、triggerSpeech()    │
│         ▼                                               │
│  ④ BanBan 宠物 UI（Electron 内加载的 HTML/Canvas，已有）   │
│     - 表情、天气、盆栽随压力值变化                         │
│     - 气泡弹出/消失                                      │
│     - 用户点击事件上报给 Electron                         │
│                                                         │
│  ⑤ Monitor 窗口（独立 OpenCV 进程，已有）                  │
│     用户点击宠物旁按钮 → Electron 启动 Monitor 进程        │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

简单说：Python 负责"看"，Electron 负责"想"（决定什么时候提醒），BanBan UI 负责"演"（展示给用户看）。

## 进程间通信

采用文件通信，简单可靠，已在原项目中验证：

```
Python 引擎 ──写入──► /tmp/banban_stress.json ──轮询读取──► Electron

格式示例（带注释）：
{
  "timestamp": "2026-04-29 14:30:00",   // 写入时间，用于判断数据新鲜度
  "stress_score": 72,                    // 0-100 压力分数
  "signals": {                           // 四个子信号，Monitor 用，宠物不直接用
    "brow_furrow": 0.8,
    "lip_press": 0.3,
    "eye_squint": 0.2,
    "expression_freeze": 0.5
  },
  "face_detected": true,                 // 是否检测到人脸
  "camera_blocked": false                // 画面是否全黑（摄像头被遮挡）
}
```

Electron 每 2 秒轮询一次这个文件，读取后驱动 BanBan UI。

## 宠物状态机

BanBan 只有两个核心状态，加一个异常处理：

```
                    ┌──────────────────┐
                    │                  │
                    ▼                  │
┌──────────┐  连续30秒超阈值  ┌──────────┐
│   待机    │ ──────────────► │   提醒    │
│  (idle)  │                 │ (alert)  │
│          │ ◄────────────── │          │
└──────────┘  用户点击气泡    └──────────┘
     ▲        或1分钟超时          │
     │        (进入5分钟冷却)       │
     │                            │
     │        ┌──────────┐        │
     └────────│  冷却中   │◄───────┘
              │(cooldown)│
              └──────────┘
               5分钟后回到待机
```

### 待机状态

- 宠物播放待机动画（呼吸、眨眼、尾巴摆动）
- 场景元素随压力值变化（表情、天气、盆栽 — 已在 UI 模块中实现）
- 不显示气泡，不主动打扰用户

### 提醒状态

触发条件：连续 30 秒（约 15 次采样）压力分数 >= 60

触发后：
- 弹出气泡，随机选择一句提醒文案
- 气泡持续最长 1 分钟
- 用户点击气泡 → 立即消除，进入冷却
- 1 分钟内未点击 → 自动消除，进入冷却

冷却期：5 分钟内不再触发提醒，即使压力持续超标。

### 异常：摄像头遮挡

当检测到画面全黑（`camera_blocked: true`）时：
- 宠物弹出一个特殊气泡："我好像看不见了，摄像头是不是被挡住了？"
- 这不算提醒状态，不受 5 分钟冷却限制
- 恢复正常后气泡自动消失

## 用户交互流程

```
用户运行 Skill
      │
      ▼
Python 检测引擎启动（后台）
      │
      ▼
Electron 启动，桌面出现 BanBan ◄── 用户看到的第一个东西
      │
      ├── 宠物安静待机，场景随压力变化
      │
      ├── 压力持续偏高 → 气泡提醒 → 用户点击或等待消失
      │
      ├── 用户点击"Monitor"按钮 → 打开 OpenCV 仪表盘窗口
      │
      └── 用户关闭宠物窗口 → 所有进程退出
```

### 宠物旁边的按钮

宠物附近有一个小按钮（或可点击区域），点击后启动/显示 Monitor 窗口。Monitor 就是原有的 OpenCV 仪表盘，展示摄像头画面、四个信号值、压力曲线等细节。

这个按钮是给"想深入了解"的用户准备的，普通用户不需要打开。

## 视觉与动画

BanBan 的视觉层已在 `files/pet-widget.html` 中实现，核心特征：

- 像素风（pixel art），8px 网格
- 低饱和度暖色系，米白 + 橘色斑纹小猫
- 透明背景，宠物"住"在桌面上
- 压力通过间接视觉变化传达（不显示数值）：
  - 低压力 → 弯弯眼微笑、太阳白云、盆栽开花
  - 高压力 → 皱眉、乌云下雨、盆栽枯萎
- 夜间自动切换为蜷睡状态（不打扰用户）

### Demo 阶段的动画范围

- 待机：呼吸、眨眼、尾巴摆动（已有）
- 提醒：弹出气泡 + 现有的情绪表现变化（已有）
- 点击反馈：歪头 + 飘爱心（已有）

后续可扩展：跳跃挥手动画、休眠子状态、更多角色动作。形象本身也可以后续替换，不影响产品逻辑。

## 提醒文案

Demo 阶段只做一种提醒模式，配十几句随机文案。文案原则：

- 短（不超过 20 字）
- 温暖（"要不"、"记得"、"先...吧"，不用命令式）
- 具体（喝水、伸懒腰、看远处，不说"你要放松"这种抽象建议）

文案按压力档位分组，已在 UI 模块中实现。Electron 层可通过 API 覆盖或扩展。

## 技术选型

| 层 | 技术 | 状态 |
|---|---|---|
| 情绪检测 | Python + MediaPipe + OpenCV | 已有 |
| 宿主框架 | Electron（透明窗口 + 置顶 + 点击穿透）| 待建 |
| 宠物 UI | HTML Canvas 像素绘制 | 已有 |
| 进程通信 | JSON 文件（/tmp/banban_stress.json）| 沿用已有方案 |
| Monitor | OpenCV 窗口（原有仪表盘）| 已有 |

### 为什么选 Electron

- BanBan UI 是 HTML/Canvas，需要一个能承载 Web 内容的桌面容器
- macOS 上透明窗口 + 置顶 + 点击穿透，Electron 最成熟
- 集成指南中已有 Electron 的参考代码
- 黑客松阶段优先跑通，包体大小不是问题

---

## 需要新建的部分

已有的两个模块（Python 检测引擎 + BanBan UI）基本不需要改动，核心工作是建 Electron 中间层：

1. **Electron 主进程** — 创建透明置顶窗口，加载 pet-widget.html，处理点击穿透
2. **轮询桥接层** — 每 2 秒读取 JSON 文件，调用 `PetWidget.setStress()` 驱动 UI
3. **提醒状态机** — 实现"连续 30 秒超阈值 → 触发 → 1 分钟超时 → 5 分钟冷却"的逻辑
4. **Monitor 启动器** — 点击按钮时启动 Python Monitor 进程
5. **摄像头遮挡检测响应** — 读取 `camera_blocked` 字段，触发特殊气泡
6. **Python 引擎小改** — 输出格式增加 `face_detected` 和 `camera_blocked` 字段，增加画面亮度检测

## 启动与退出

### 启动流程

Skill 被调用时：
1. 启动 Python 检测引擎（后台进程）
2. 启动 Electron 应用（前台，显示 BanBan）
3. Electron 开始轮询 JSON 文件

### 退出流程

用户关闭 BanBan 窗口时：
1. Electron 退出
2. 同时终止 Python 检测引擎进程
3. 清理 /tmp 下的 JSON 文件

## 不做什么（Demo 阶段）

- 不做历史数据存储和趋势分析
- 不做用户设置界面（阈值、冷却时间等暂时硬编码）
- 不做多显示器适配
- 不做自动更新
- 不做 Windows/Linux 适配
- 不做精细化的多阶段提醒（只有一种提醒模式）
- 不做宠物拖拽移动（固定位置）
